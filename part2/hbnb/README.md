# HBnB - Business Logic and API (Part 2)

## Description

This directory contains the second phase of the HBNB project, focusing on the implementation of the Business Logic and Presentation layers. Built with **Python, Flask, and Flask-RESTx**, this phase brings the documented architecture to life by establishing the core models and RESTful API endpoints needed to manage Users, Places, Reviews, and Amenities.

To ensure modularity and separation of concerns, the application utilizes the **Facade design pattern** to bridge the API and Business Logic layers. At this stage, the Persistence layer relies on an **In-Memory Repository** for object storage and validation, paving the way for a seamless transition to a database-backed SQLAlchemy setup in the future.

This project was developed collaboratively by **Abdulrhman Alsalhi**, **Osama Alhamdan**, and **Eman Hamdan** as part of the Software Engineering curriculum at **Holberton School by Tuwaiq Academy**.

## API Endpoints Summary

The API handles CRUD operations for the core entities. Note that user authentication and role-based access control are deferred to Part 3.

| Entity | Supported Methods | Description & Constraints |
| --- | --- | --- |
| **Users** | `GET`, `POST`, `PUT` | Manages user accounts. Passwords are excluded from GET responses. *DELETE is not supported.* |
| **Amenities** | `GET`, `POST`, `PUT` | Manages property amenities. *DELETE is not supported.* |
| **Places** | `GET`, `POST`, `PUT` | Manages rental places. Includes validation for price, latitude, and longitude. Responses return associated owner and amenity data. *DELETE is not supported.* |
| **Reviews** | `GET`, `POST`, `PUT`, `DELETE` | Manages place reviews. Validates review text and ensures association with a valid User and Place. Includes full deletion support. |

## Environment and Installation

### Requirements

* Python 3.8+
* Flask
* Flask-RESTx

### Setup

1. Clone the repository and navigate to the project directory:
```bash
cd part2/hbnb

```


2. Install the required dependencies:
```bash
pip install -r requirements.txt

```


3. Run the Flask application:
```bash
python3 run.py

```



---

## Testing and Validation

All endpoints have been rigorously tested to validate input formats, attribute rules, and correct HTTP status code returns.

To run the automated test suite, use the following command from the root project directory:

```bash
python3 -m unittest discover tests

```

Manual black-box testing can also be performed via **cURL**, Postman, or by interacting with the Swagger documentation automatically generated by Flask-RESTx at the API's root URL.
